# 02 — User Flows

## Fluxo 1 — Produzir relatório técnico completo

1. Usuário informa o caso.
2. Usuário envia documentos, dados ou descrição.
3. Frankbot identifica o objetivo da análise.
4. Frankbot classifica o tipo de demanda.
5. Frankbot ativa as skills necessárias.
6. Frankbot extrai fatos relevantes.
7. Frankbot identifica lacunas.
8. Frankbot organiza achados.
9. Frankbot elabora recomendações.
10. Frankbot redige relatório técnico.
11. Frankbot executa QA adversarial.
12. Frankbot entrega versão final ou versão com ressalvas.

## Fluxo 2 — Produzir síntese executiva

1. Usuário envia relatório ou contexto longo.
2. Frankbot identifica público destinatário.
3. Frankbot extrai problema, risco, impacto e providência.
4. Frankbot gera resumo curto para tomada de decisão.

## Fluxo 3 — Revisar texto produzido por humano

1. Usuário envia minuta.
2. Frankbot avalia clareza, cautela, estrutura e risco de afirmações indevidas.
3. Frankbot sugere ajustes.
4. Frankbot entrega versão revisada.

## Fluxo 4 — Transformar caso em requisição de informações

1. Usuário descreve lacuna, dúvida ou necessidade de comprovação.
2. Frankbot identifica quais informações faltam.
3. Frankbot estrutura requisição objetiva ao órgão responsável.
4. Frankbot evita tom acusatório.
5. Frankbot indica documentos ou evidências esperadas.

## Fluxo 5 — Evoluir biblioteca de skills

1. Usuário solicita uma tarefa nova.
2. Frankbot compara a tarefa com o catálogo de skills existentes.
3. Se houver skill adequada, usa a skill.
4. Se não houver, propõe ou cria nova skill.
5. Ao final da execução, revisa se a skill usada foi suficiente.
6. Se houver lacuna, registra melhoria recomendada.
