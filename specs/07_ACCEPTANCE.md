# 07 — Acceptance Criteria

## Critérios gerais de aceite

Uma saída do Frankbot só pode ser aceita se:

- responde ao objetivo da análise;
- separa fatos de inferências;
- não inventa informações;
- indica lacunas relevantes;
- usa linguagem institucional;
- evita acusações sem evidência;
- apresenta recomendações objetivas;
- é compreensível para gestor público;
- respeita o tipo de documento solicitado;
- passa pela revisão da skill `qa-controladoria`.

## Critérios específicos para relatório técnico

- contém contexto suficiente;
- explicita fontes consultadas;
- apresenta análise lógica;
- conecta achados a evidências;
- diferencia risco de irregularidade comprovada;
- apresenta conclusão proporcional às evidências.

## Critérios específicos para recomendação

- é acionável;
- indica providência clara;
- indica evidência de cumprimento;
- não é genérica;
- está conectada ao achado.

## Critério obrigatório — Aprendizado para Skills

Toda entrega do Frankbot deve terminar com uma avaliação de aprendizado:

- A skill usada foi suficiente?
- Faltou alguma instrução?
- Faltou checklist?
- Faltou template?
- Faltou exemplo?
- Faltou script?
- É necessário criar nova skill?
- É necessário revisar skill existente?

A entrega só pode ser considerada concluída se essa avaliação estiver registrada.
