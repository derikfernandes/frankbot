# 09 — Decisions

## Decisão 001 — Frankbot será skill-centric

O Frankbot será organizado a partir de Agent Skills, e não como um prompt único gigante.

Justificativa: o Frank domina várias habilidades; portanto, a melhor forma de modelar seu trabalho é separar essas habilidades em competências reutilizáveis.

Status: decidido.

## Decisão 002 — Relatórios devem usar linguagem cautelosa

O Frankbot deve sempre diferenciar evidência, inferência, risco e conclusão.

Justificativa: a Controladoria precisa atuar com precisão técnica e responsabilidade institucional.

Status: decidido.

## Decisão 003 — A versão inicial não terá integração automática com sistemas internos

A primeira versão trabalhará com documentos e textos fornecidos pelo usuário.

Justificativa: reduzir complexidade inicial e validar o padrão de produção dos relatórios antes das integrações.

Status: decidido.

## Decisão 004 — Toda saída importante terá QA adversarial

Antes de aceitar uma entrega, o Frankbot deve passar pela skill `qa-controladoria`.

Justificativa: relatórios institucionais exigem controle de qualidade e cautela.

Status: decidido.

## Decisão 005 — Evolução contínua das Skills

O Frankbot deve sempre avaliar, durante e após cada tarefa, se as skills existentes foram suficientes.

Quando uma demanda exigir um procedimento novo, recorrente ou especializado, o agente deve propor ou criar uma nova skill.

Quando uma skill existente for usada e apresentar lacunas, o agente deve propor sua revisão.

Status: decidido.
