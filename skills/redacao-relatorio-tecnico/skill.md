# Objetivo

Produzir relatório técnico claro, cauteloso, institucional e útil para tomada de decisão.

# Procedimento

1. Identificar o objetivo do relatório.
2. Identificar o público destinatário.
3. Separar fatos, evidências, inferências e recomendações.
4. Organizar o relatório em seções lógicas.
5. Redigir com linguagem institucional.
6. Indicar limitações da análise.
7. Evitar afirmações não sustentadas.
8. Encaminhar para QA antes da entrega final.
9. Ao final, avaliar se esta skill precisa ser revisada.

# Estrutura padrão

1. Título
2. Objeto
3. Contextualização
4. Objetivo da análise
5. Fontes consultadas
6. Análise técnica
7. Achados ou pontos de atenção
8. Recomendações
9. Conclusão
10. Limitações

# Regras de cautela

Não usar, salvo se comprovado por documento competente:

- fraude comprovada;
- má-fé;
- crime;
- irregularidade comprovada;
- responsável pelo dano.

Preferir:

- indício;
- possível inconsistência;
- necessidade de apuração complementar;
- fragilidade de controle;
- risco administrativo;
- recomenda-se verificação.

# Critérios de qualidade

O relatório deve:

- responder ao objetivo;
- ser compreensível para gestor público;
- conter evidências;
- indicar lacunas;
- ser proporcional às informações disponíveis;
- gerar encaminhamento prático.
